# 模型目录 V2 一次性迁移方案

> 状态：方向已确认，尚未实施数据库与 UI cutover。
>
> 目标：模型只分为内置与自定义两类，停止上游模型同步，以数据库中的模型、价格、路由和提示词策略作为唯一事实来源。

## 1. 最终约束

1. 模型来源只有 `builtin` 和 `custom`。
2. 不再请求或合并任何上游 `/models`、Codex model list 或供应商模型目录。
3. Codex 常用模型由版本化数据库 seed 提供；升级只插入缺失项，不覆盖用户修改。
4. 其他模型由用户手工新增，或从本地文件预览后导入；导入不访问网络。
5. 每个模型直接配置输入、缓存输入、输出价格，运行时计费只读取数据库。
6. 模型保存、价格保存、路由保存必须在一个事务中完成，不能出现“模型成功、价格失败”的半成品。
7. 模型不携带官方人格提示词。客户端非空提示词默认原样传递。
8. 只有实际选中的 Codex backend 要求非空 `instructions` 时，传输层才补中性最小值：

   ```text
   Follow the user's instructions.
   ```

9. 模型可显式选择 `passthrough`、`fallback`、`override` 三种 instructions 策略。
10. 账号池和聚合 API 的请求体必须在实际候选确定后分别改写，禁止跨供应商复用 Codex 专用 body。

## 2. 为什么不继续扩展旧模型目录

当前模型域同时存在目录、价格规则、来源模型、来源映射、映射偏好、供应商模型、额度分配和 API 级模型覆盖。读取模型时还会触发路由补建和数据库写入，远端发现又会自动创建目录、映射和零价格规则。

这些结构存在三个根本问题：

- 同一件事有多份状态，切换账号或聚合 API 后容易留下陈旧路由。
- Gateway 实际没有消费全部管理映射，管理页显示的配置不一定等于真实转发行为。
- 模型和价格分开保存，计费规则又同时来自编译期 seed 与数据库，无法形成稳定审计链。

V2 不做双写兼容层。新版本一次迁移到新表，运行时只读写 V2；旧表保留一个版本作为无流量 smoke 窗口的恢复与审计材料，随后删除。

## 3. 数据模型

### 3.1 `models`

```text
id                              TEXT PRIMARY KEY
slug                            TEXT NOT NULL UNIQUE COLLATE NOCASE
display_name                    TEXT NOT NULL
provider                        TEXT
family                          TEXT
category                        TEXT
tags_json                       TEXT NOT NULL DEFAULT '[]'
origin                          TEXT NOT NULL CHECK (origin IN ('builtin', 'custom'))
enabled                         INTEGER NOT NULL DEFAULT 1
supported_in_api                INTEGER NOT NULL DEFAULT 1
context_window                  INTEGER
default_reasoning_effort        TEXT
capabilities_json               TEXT NOT NULL DEFAULT '{}'
instructions_mode               TEXT NOT NULL DEFAULT 'passthrough'
                                CHECK (instructions_mode IN ('passthrough', 'fallback', 'override'))
instructions_text               TEXT
builtin_revision                INTEGER
created_at                      INTEGER NOT NULL
updated_at                      INTEGER NOT NULL
```

约束：

- `override` 必须有非空 `instructions_text`，由存储层和 RPC 同时校验。
- `fallback` 可以暂时没有文本；此时非 Codex 上游保持空，Codex 上游最后使用传输层最小兜底。
- `builtin` 不允许物理删除，只允许禁用，避免后续 seed 将用户删除的模型重新插入。
- 内置模型允许用户编辑；版本升级不得覆盖已有行的显示名、能力、价格、提示词或启用状态。
- 不把旧 `base_instructions` 自动迁移为 `instructions_text`。前者是远端目录元数据，不代表用户主动配置的提示词策略。

### 3.2 `model_prices`

```text
model_id                        TEXT PRIMARY KEY REFERENCES models(id) ON DELETE CASCADE
currency                        TEXT NOT NULL DEFAULT 'USD' CHECK (currency = 'USD')
input_microusd_per_1m           INTEGER
cached_input_microusd_per_1m    INTEGER
output_microusd_per_1m          INTEGER
price_status                    TEXT NOT NULL
                                CHECK (price_status IN ('official', 'estimated', 'custom', 'missing'))
price_source                    TEXT
created_at                      INTEGER NOT NULL
updated_at                      INTEGER NOT NULL
CHECK (input_microusd_per_1m IS NULL OR input_microusd_per_1m >= 0)
CHECK (cached_input_microusd_per_1m IS NULL OR cached_input_microusd_per_1m >= 0)
CHECK (output_microusd_per_1m IS NULL OR output_microusd_per_1m >= 0)
CHECK (
    (price_status = 'missing'
        AND input_microusd_per_1m IS NULL
        AND cached_input_microusd_per_1m IS NULL
        AND output_microusd_per_1m IS NULL)
    OR
    (price_status <> 'missing'
        AND input_microusd_per_1m IS NOT NULL
        AND cached_input_microusd_per_1m IS NOT NULL
        AND output_microusd_per_1m IS NOT NULL)
)
```

价格使用整数 micro-USD，禁止用 `REAL` 参与最终扣费。`1 USD = 1,000,000 micro-USD`。

`price_status=missing` 的模型可以保存和测试路由，但不能进入需要钱包扣费的权限组；请求返回明确的 `model_price_missing`，不能按零价格放行。

### 3.3 `model_routes`

```text
id                              TEXT PRIMARY KEY
model_id                        TEXT NOT NULL REFERENCES models(id) ON DELETE CASCADE
source_kind                     TEXT NOT NULL CHECK (source_kind IN ('account_pool', 'aggregate_api'))
source_id                       TEXT NOT NULL
upstream_model                  TEXT NOT NULL
enabled                         INTEGER NOT NULL DEFAULT 1
priority                        INTEGER NOT NULL DEFAULT 0
weight                          INTEGER NOT NULL DEFAULT 1
created_at                      INTEGER NOT NULL
updated_at                      INTEGER NOT NULL
UNIQUE(model_id, source_kind, source_id, upstream_model)
```

路由规则：

- 账号不再各自复制一份模型行。账号来源使用池级路由，例如 `source_kind=account_pool`、`source_id=default`。
- 聚合 API 路由必须显式选择 `aggregate_api_id`，并允许配置不同的 `upstream_model`。
- Gateway 只消费 `model_routes`，管理页和真实转发不再使用两套语义。
- 候选选择只读路由，不在请求路径中补建模型、来源或映射。
- 删除或禁用来源时，用单个事务禁用对应路由；请求读取过程不得写数据库。

### 3.4 权限组

保留现有模型权限组概念，但关联改为 `model_id`。

默认组使用查询语义 `all_enabled_models`，不为每个新内置模型复制关联行。自定义组才维护显式模型集合。倍率继续使用整数 `rate_multiplier_millis`。

### 3.5 `request_charge_snapshots`

```text
request_log_id                  INTEGER PRIMARY KEY REFERENCES request_logs(id) ON DELETE RESTRICT
model_id                        TEXT
model_slug                      TEXT NOT NULL
usage_source                    TEXT NOT NULL CHECK (usage_source IN ('actual', 'estimated'))
input_tokens                    INTEGER NOT NULL
cached_input_tokens             INTEGER NOT NULL
output_tokens                   INTEGER NOT NULL
input_microusd_per_1m           INTEGER NOT NULL
cached_input_microusd_per_1m    INTEGER NOT NULL
output_microusd_per_1m          INTEGER NOT NULL
rate_multiplier_millis          INTEGER NOT NULL
base_cost_microusd              INTEGER NOT NULL
charged_cost_microusd           INTEGER NOT NULL
currency                        TEXT NOT NULL
created_at                      INTEGER NOT NULL
```

该表不可随模型价格更新。请求完成时保存实际使用的 token、单价和倍率快照，保证历史账单可复算。

## 4. 计费公式

```text
uncached_input = max(input_tokens - cached_input_tokens, 0)

numerator =
    uncached_input      × input_microusd_per_1m
  + cached_input_tokens × cached_input_microusd_per_1m
  + output_tokens       × output_microusd_per_1m

base_cost_microusd = ceil(numerator / 1,000,000)

charged_cost_microusd =
    ceil(numerator × rate_multiplier_millis / 1,000,000,000)
```

实现要求：

- 中间计算使用 `i128`，落库前做范围检查。
- 三项合计并应用倍率后只做一次最终取整；`base_cost_microusd` 仅作原价快照，不作为倍率计算的中间整数，避免二次向上取整。
- 归一化后的 `input_tokens` 表示包含缓存命中在内的总输入 token，`cached_input_tokens` 是其子集。如上游分别返回未缓存与缓存 token，适配器必须先求和成总输入，禁止各协议直接套用不同口径。
- 优先使用上游返回的 actual usage。
- 上游缺失 usage 或流式请求提前断开时，使用确定性的本地估算并记录 `usage_source=estimated`；不得静默按 0 token 扣费。
- 价格解析、快照写入、钱包流水和余额更新在同一事务完成。
- 钱包幂等键使用请求 ID，重试或重复终态事件不得重复扣费。

## 5. Instructions 策略

策略在协议适配完成后、实际上游候选的 transport rewrite 之前执行一次。

多入口统一按以下顺序归一化：

1. Anthropic `system`、Gemini `systemInstruction` 和 Chat/Responses 的 system/developer 内容先映射为有序的客户端 instruction segments，文本内容不做 `trim`。
2. 顶层 `instructions` 非空时保留其原值，不提升或删除同时存在的 leading system/developer 消息；两者都计为客户端已传内容。
3. 顶层缺失或纯空白、且存在连续的 leading 文本 system/developer 消息时，按原顺序用 `\n\n` 连接后提升，只移除已提升的 leading 项。
4. 结构化非文本 system block 应由 typed 协议适配器保真处理，不得因为无法提取字符串就视为“客户端未传”。目标协议无法无损表示时返回明确验证错误，不静默丢弃或另外注入 fallback。

### `passthrough`

- 非空客户端 `instructions` 原样保留，包括首尾空格和换行。
- Anthropic `system`、Gemini `systemInstruction` 只做字段/角色映射，不生成或追加人格文本。
- 客户端缺失或为空时不使用模型文本。

### `fallback`

- 任一已归一化的客户端 instruction channel 存在非空内容时都优先，绝不覆盖。
- 客户端缺失、`null`、空串或纯空白时使用模型的 `instructions_text`。
- 模型文本也为空时，非 Codex 上游保持为空；实际 Codex backend 最后使用中性 transport fallback。

### `override`

- 使用模型 `instructions_text` 替换规范化后的顶层 instruction channel。
- 覆盖范围包括顶层 `instructions` 和所有连续的、可提升的 leading system/developer 内容；这些 leading 项在写入模型文本后移除，避免旧内容仍被上游执行。
- 对话历史中间位置的 system/developer message 仍视为历史消息，不在 V2 中静默删除；UI 必须明确显示这一边界。
- 不提供隐式 `append` 模式，避免顺序、分隔符和重复注入语义不确定。

默认值：所有内置与新建自定义模型均为 `passthrough`、`instructions_text=NULL`。中性 Codex transport fallback 不存入模型行，因此不是模型的“官方默认提示词”。

## 6. 内置模型 seed

首版 seed 不包含 `auto`，包括：

```text
gpt-5
gpt-5-codex
gpt-5-codex-mini
gpt-5.1
gpt-5.1-codex
gpt-5.1-codex-max
gpt-5.1-codex-mini
gpt-5.2
gpt-5.2-codex
gpt-5.3-codex
gpt-5.4-mini
gpt-5.4
```

空库和后续版本的 seed 使用 `INSERT OR IGNORE`。新版本可以插入新模型；对已存在模型及价格行（包括 `price_status=missing`）均不做隐式 `UPDATE`。已验证的新价格只能为“完全没有价格行”的内置模型补行；把已有 `missing` 改成有价必须由用户在 UI 明确确认，避免升级改变计费。内置价格从当前已验证的数据库/编译期 price seed 固化到迁移数据；无法确认的价格标为 `missing`，不能猜测。

## 7. 本地导入

本地导入流程：

1. 读取用户选择的 JSON 文件。
2. 只解析允许字段，拒绝未知策略值、负价格和无效路由引用。
3. 返回 preview：新增、更新、冲突、跳过和错误列表。
4. 用户确认后在一个事务中写入。
5. 导入项统一标记为 `custom`，不得伪造 `builtin`。

冲突策略只提供 `keep_existing` 和 `replace_custom`。内置模型不能被批量导入覆盖；用户可在模型编辑页单独修改。

## 8. RPC 与前端边界

新增聚合 RPC：

- `managed_model_list_v2`
- `managed_model_get_v2`
- `managed_model_upsert_v2`
- `managed_model_delete_v2`
- `managed_model_import_preview_v2`
- `managed_model_import_commit_v2`

`managed_model_upsert_v2` 的 payload 同时包含 model、price、routes 和 permission groups，服务端统一校验并事务提交。

前端模型编辑器只保留四个区域：

1. 基本信息与能力。
2. 输入/缓存输入/输出价格。
3. 账号池与聚合 API 路由。
4. Instructions 策略与文本。

删除“远端刷新”“同步供应商模型”“自动合并目录”按钮。Codex `models_cache.json` 不再自动写入，只保留用户主动触发的本地导出。

## 9. 一次性 cutover

### 9.1 迁移前

1. 进入维护态：停止接受新 Gateway 请求（返回可重试的 503），暂停后台同步/计费任务，排空在途请求和钱包事务。
2. 获取数据库旁路径的跨进程独占迁移锁，并校验没有其他 desktop/service-mode 进程持有旧 schema 连接；无法保证单写者时拒绝迁移。该锁必须从备份前持有到 smoke test 结束。
3. 使用 SQLite online backup API 创建带时间戳备份，不能直接复制仍有 WAL 的数据库文件。
4. 在同一维护窗口执行只读 preflight 并输出报告：重复 slug、负价格、孤儿映射、无来源模型、无价格的付费模型。
5. 任一不可自动解决的错误阻止升级，不修改原库。

### 9.2 单事务迁移

1. 创建 V2 表、外键、CHECK 和索引。
2. 构建确定性 staging set：对 seed 清单中与旧数据同 slug 的模型，复用同一个 `builtin` 目标行；seed 提供默认值，旧库明确 `user_edited=1` 的字段、禁用状态和已验证显式价格优先。普通 remote metadata 不得覆盖 seed。
3. 插入 staging 中的内置模型，再用 `INSERT OR IGNORE` 补齐真正缺失的 seed；不得因 slug 唯一键冲突跳过用户修改。
4. 将旧目录中 `source_kind=custom` 或 `user_edited=1` 的非内置 slug 迁为 `custom`。
5. 将旧 exact price rule 或当前已解析价格固化为每模型价格；pattern rule 只用于迁移时解析，不进入 V2 runtime。
6. 迁移有效显式聚合路由；账号来源折叠为单个 `account_pool/default` 路由。
7. 迁移权限组和倍率。
8. 校验所有外键、唯一键、价格范围和默认组语义。
9. 写入 V2 schema marker 后提交。

迁移失败时整个事务回滚，保留备份与原表。

事务提交后仍不立即恢复流量：在持有迁移锁的无流量窗口执行 schema invariant、模型读取、路由 dry-run 和计费 dry-run smoke test。smoke test 失败时关闭所有连接并从 online backup 恢复；只有通过后才释放锁并重新开放 Gateway。

### 9.3 代码切换

- 新版本只读写 V2，不双写旧表。
- 旧表在一个版本内保留只读，仅用于无流量 smoke 窗口内的恢复。Gateway 重新开放后 V2 已可能产生新模型编辑、请求日志和钱包流水，此时禁止自动降级或恢复旧备份，只允许 forward-fix 或经人工审计的 V2 导出/replay。下一次 schema 清理再删除旧表。
- 所有 SQLite 连接统一执行 `PRAGMA foreign_keys=ON`。
- 启动后执行轻量 invariant check；发现损坏时拒绝开启 Gateway 写入能力并给出恢复提示。

## 10. 必须删除的旧运行时行为

- 上游 `/models` 获取、定时刷新和远端 merge。
- `source_kind=remote`、远端 prune 和 user-edited/remote 冲突合并。
- 聚合 API 发现后自动创建平台模型、映射和零价格规则。
- 读取模型或处理请求时 bootstrap/reconcile 并写数据库。
- 每个账号复制模型来源行的逻辑。
- 运行时编译期 `PRICE_SEEDS` fallback。
- 自动写 Codex `models_cache.json`。
- 前端不一致的 `managed-model-routing` / `model-routing` query key。

供应商连通性测试仍可保留，但只能测试用户已配置的模型，不能顺便发现或同步模型。

## 11. 验收矩阵

### 数据与迁移

- 空库、现有生产库和包含脏旧映射的库都能得到确定性结果。
- 重复运行迁移不重复 seed、不覆盖用户编辑。
- 内置禁用后升级不会重新启用。
- migration failure 后原库可继续由旧版本读取。
- 成功提交后的无流量 smoke test 失败可完整恢复备份；重新开放流量后会拒绝降级启动。
- upsert/import 在 model、price、route 和 permission group 任一写入阶段故障注入时均整单回滚。

### Gateway

- `/v1/responses`、`/v1/chat/completions` 的流式、非流式、tools、tool_calls 均覆盖。
- 账号切换共享 Codex transport profile，不重复生成模型状态。
- 聚合候选 Codex → MiniMax/Claude 和反向切换时，每个 attempt 独立生成 body。
- 非 Codex 候选不会收到中性 Codex fallback。
- Anthropic/Gemini 客户端非空 system 文本在字段映射后内容保持一致。
- 真实 Codex target 的判定同时满足候选 `provider_type=codex` 且归一化后的最终 URL 处于 `/backend-api/codex`；不根据模型名、供应商显示名或单一 URL 特征猜测。
- 启动、模型编辑、路由测试和连通性测试均断言不会发出任何上游 `/models` 请求。

### Instructions

- 三种 mode × 客户端有/无内容 × 模型有/无文本 × Responses/Chat/Anthropic/Gemini × account/aggregate 表驱动覆盖。
- 顶层与 leading instruction 同时存在、空顶层+非空 leading、多文本 block 和无法无损表示的结构化 block 都有明确回归。
- missing、`null`、空串、纯空白在真实 Codex target 上得到同一中性 fallback。
- 非空内容含首尾空白时 byte-for-byte 保留。

### 计费

- 输入、缓存输入、输出分别计价。
- cached tokens 大于 input tokens 时安全夹到 0 uncached input。
- 极大 token/rate 使用 `i128` 不溢出。
- 倍率、取整、actual/estimated usage 和幂等扣费覆盖。
- 修改模型价格后，历史快照和历史钱包流水不变。
- 请求日志清理不得级联删除计费快照；快照、钱包流水与审计保留期使用同一策略。

## 12. 交付方式

实现可以拆成可审查的小提交，但只能以一个产品版本完成 cutover：

1. Core migration、V2 storage 与备份/preflight。
2. 独立 model domain、事务 RPC 与 seed/import。
3. Gateway 路由、instructions policy 和候选级 transport。
4. 计费快照与钱包原子扣费。
5. 前端模型编辑器与旧同步 UI 删除。
6. 删除旧同步 runtime，执行完整 workspace、desktop、web 和 Gateway 回归。

不引入长期 feature flag、双写或后台同步兼容层，避免新旧模型状态再次分叉。
